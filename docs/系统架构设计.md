# 系统架构设计文档

## 1. 系统总体架构

### 1.1 架构概述

本系统采用前后端分离的微服务架构，整体架构分为四层：

```
┌─────────────────────────────────────────────────────────────────┐
│                         展示层 (Presentation Layer)               │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
│  │   Web端     │  │   移动端    │  │  大屏展示   │              │
│  └─────────────┘  └─────────────┘  └─────────────┘              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                         网关层 (Gateway Layer)                    │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                    API Gateway (Nginx)                       ││
│  │         负载均衡 / 请求路由 / 认证鉴权 / 限流熔断              ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                       服务层 (Service Layer)                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │用户服务  │ │数据服务  │ │预警服务  │ │报表服务  │           │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                        │
│  │设备服务  │ │干预服务  │ │消息服务  │                        │
│  └──────────┘ └──────────┘ └──────────┘                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    AI服务层 (AI Service Layer)                    │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                    模型推理服务                              ││
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       ││
│  │  │风险评估  │ │异常检测  │ │趋势预测  │ │行为识别  │       ││
│  │  │  模型    │ │  模型    │ │  模型    │ │  模型    │       ││
│  │  └──────────┘ └──────────┘ └──────────┘ └──────────┘       ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                       数据层 (Data Layer)                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │PostgreSQL│ │  Redis   │ │ 时序数据库 │ │ 文件存储 │           │
│  │ (主库)   │ │ (缓存)   │ │ (InfluxDB)│ │ (MinIO) │           │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘           │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue.js 3 + Element Plus + ECharts | 响应式UI框架，丰富的图表组件 |
| 后端框架 | FastAPI | 高性能异步框架，自动API文档 |
| ORM | SQLAlchemy 2.0 | 异步ORM，支持复杂查询 |
| 深度学习 | PyTorch 2.0 | 灵活的模型构建与训练 |
| 主数据库 | PostgreSQL 15 | 关系型数据库，支持JSON |
| 缓存 | Redis 7.0 | 高性能缓存，消息队列 |
| 时序数据库 | InfluxDB 2.0 | 存储健康指标时序数据 |
| 文件存储 | MinIO | 对象存储，存储模型文件 |
| 消息队列 | Redis Streams | 异步任务处理 |
| 容器化 | Docker + Docker Compose | 简化部署与运维 |

## 4. 接口设计规范

### 4.1 RESTful API规范

- 基础路径: `/api/v1`
- 认证方式: JWT Bearer Token
- 响应格式: JSON

### 4.2 统一响应格式

```json
{
    "code": 200,
    "message": "success",
    "data": {},
    "timestamp": "2024-01-01T00:00:00Z"
}
```

### 4.3 错误响应格式

```json
{
    "code": 400,
    "message": "参数错误",
    "detail": "心率值超出正常范围",
    "timestamp": "2024-01-01T00:00:00Z"
}
```
