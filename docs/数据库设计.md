# 数据库设计文档

## 1. 数据库概述

本系统采用多数据库架构：
- **PostgreSQL**: 存储关系型数据（用户、老人信息、预警记录等）
- **Redis**: 缓存与会话管理
- **InfluxDB**: 存储时序健康数据

---

## 3. InfluxDB 时序数据库设计

### 3.1 健康指标时序数据 (health_metrics_ts)

**Measurement**: health_metrics

**Tags**:
- elderly_id: 老人ID
- metric_code: 指标编码
- device_id: 设备ID

**Fields**:
- value: 指标值 (float)
- quality: 数据质量评分 (int)

**Timestamp**: 数据采集时间

### 3.2 活动数据时序数据 (activity_data_ts)

**Measurement**: activity_data

**Tags**:
- elderly_id: 老人ID
- activity_type: 活动类型

**Fields**:
- duration: 持续时间 (int, 秒)
- intensity: 强度 (float)
- steps: 步数 (int)

**Timestamp**: 活动记录时间

## 4. Redis 缓存设计

### 4.1 缓存键命名规范

| 缓存类型 | 键名格式 | 过期时间 | 说明 |
|----------|----------|----------|------|
| 用户会话 | session:{user_id} | 24h | 用户登录会话 |
| 用户权限 | permission:{user_id} | 1h | 用户权限缓存 |
| 老人信息 | elderly:{elderly_id} | 30min | 老人基本信息 |
| 实时健康数据 | realtime:health:{elderly_id} | 5min | 最新健康数据 |
| 预警计数 | alert:count:{elderly_id} | 1h | 预警数量统计 |
| 模型预测结果 | prediction:{elderly_id} | 10min | 模型预测缓存 |

### 4.2 消息队列设计

使用 Redis Streams 实现消息队列：

- **health_data_stream**: 健康数据流
- **alert_stream**: 预警消息流
- **notification_stream**: 通知消息流
